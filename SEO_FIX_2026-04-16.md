# GSC Canonical Cleanup — 2026-04-16

**Status:** Commit `abd401d` is on local `main`. Needs `git push origin main` from your machine to trigger Vercel auto-deploy.

## What was broken

Three compounding causes drove the ~963 "Duplicate, Google chose different canonical" errors in GSC:

1. **All 250 industry pages rendered identical copy.** `IndustryVerticalClient.tsx` had `t('industries.medical-devices.X')` hardcoded throughout the UI — every `/locations/{city}/industries/{industry}` page showed the same medical-devices strings regardless of the `industry` param. Google saw them as duplicates and picked one canonical arbitrarily.
2. **Contradictory noindex + canonical signals.** The 8 machine-translated locales (`fr/de/it/pt/ru/ja/zh/ko`) were served with `robots: { index: false, follow: true }` AND a cross-locale canonical pointing to `/en/`. Google ignores canonical on noindex pages, so each noindex locale was effectively invisible while still being crawled — the canonical signal was wasted.
3. **Hreflang advertised the noindex URLs.** The sitemap's hreflang alternates listed all 10 locales, telling Google to keep crawling URLs whose canonical it was simultaneously being told to ignore.

## What shipped in `abd401d`

Strategy: **kill the 8 deprecated locales entirely**, so we serve only `/en/` and `/es/`.

| Area | Change |
|---|---|
| `middleware.ts` | 301-redirect `/{fr,de,ja,zh,ko,it,pt,ru}/**` → `/en/**`, preserving path + query |
| `app/constants/seo-config.ts` | `LOCALES = ['en','es']`; `NOINDEX_LOCALES = ∅` (kept as empty Set for backward-compat imports); new `DEPRECATED_LOCALES` export for bookkeeping |
| `app/sitemap.ts` | Only `en` + `es` `<loc>` entries; hreflang alternates limited to `en`, `es`, `x-default`; `lastModified` bumped to `2026-04-16` to force recrawl |
| `app/[lang]/locations/[city]/industries/[industry]/IndustryVerticalClient.tsx` | Replaced hardcoded `t('industries.medical-devices.X')` with dynamic `` t(`industries.${industry}.X`) `` — 250 pages now render their own copy |
| `app/context/LanguageContext.tsx` + `components/Navbar.tsx` | Language picker + detection reduced to `en`/`es` only; geolocation fallback collapsed to `en` default |
| 25 `page.tsx` / `layout.tsx` files under `app/[lang]/` | Removed the `NOINDEX_LOCALES.has(lang)` branch from every `generateMetadata`; canonical now always points to self; hreflang reduced to `en`, `es`, `x-default` |

`npx tsc --noEmit` passes clean. Local Turbopack build blocked only by sandbox network restriction on Google Fonts — builds fine on Vercel.

## GSC actions after deploy

1. **Resubmit sitemap** — `Settings → Sitemaps → Add: https://nearshorenavigator.com/sitemap.xml` (forces a fresh read of the updated one).
2. **URL Inspection + Request Indexing** — for the highest-priority pages:

   - `/en/locations/tijuana/industries/medical-devices`
   - `/en/locations/tijuana/industries/automotive`
   - `/en/services/contract-manufacturing-tijuana`
3. **Don't** manually request indexing for the `/fr /de /ja /zh /ko /it /pt /ru` URLs — they're now 301s. Google will drop them from the index organically once recrawled.
